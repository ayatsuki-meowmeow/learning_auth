認証・認可の実装をセキュリティ観点でレビューしてください。

## 調査対象

- `openapi.yaml` — 認証エンドポイント定義
- `apps/api/src/db/schema/` — ユーザー・セッション・トークンのテーブル定義
- `apps/api/src/routes/` — 認証ルート
- `apps/api/src/` — JWT 検証ミドルウェア
- `apps/api/src/crud/` — 認証関連 DB 操作
- `packages/schema/src/` — 認証リクエストの Zod スキーマ

## レビュー観点

| カテゴリ | チェック内容 |
|---------|------------|
| パスワード | argon2 / bcrypt の使用・ストレッチング回数 |
| JWT | 有効期限・署名アルゴリズム・秘密鍵の環境変数管理 |
| タイミング攻撃 | 定数時間比較の使用 |
| レート制限 | ブルートフォース対策 |
| エラーレスポンス | ユーザー存在確認に使える情報の漏洩有無 |
| Cookie | HttpOnly・Secure・SameSite 属性 |
| 認可 | ミドルウェアの認証確認・他ユーザーリソースへの不正アクセス |

## 出力フォーマット

🔴 重大 / 🟡 要改善 / 🔵 改善推奨 で分類し、問題・なぜ危険か・修正方法を示す。

$ARGUMENTS が指定された場合はそのファイル・トピックに絞ってレビューする。
